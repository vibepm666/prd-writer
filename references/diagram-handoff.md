# 图表职责划分

PRD 中的图用于 **对齐流程与结构**，不替代专业制图交付物。

## 用户动线（§3 核心用户动线）

| 复杂度 | 处理方式 |
| --- | --- |
| **简单主流程**（单角色、≤15 节点、无泳道） | PRD 内嵌 **Mermaid** `flowchart`，含主流程 + 1–2 条异常分支 |
| **复杂流程**（多角色泳道、跨系统、需插图进汇报/PRD PDF） | 调用 **`diagram-generator`** 技能渲染 PNG，PRD 内写：`![流程图](docs/images/xxx.png)` 并链回源 XML |
| **状态机**（单功能多状态切换） | 优先 Mermaid `stateDiagram-v2`；节点过多时用表格 + 简短 Mermaid |

## 页面骨架（§4.1）

- 用 **ASCII 线框图**，选 **1 个** 最核心页面
- 体现：导航方向、区域划分、视觉重心、覆盖层（弹窗/抽屉）
- 示例见 `prd-template.md` §4.1

## 禁止

- 在 PRD 里用纯文字长篇描述本可用一张流程图说清的主路径
- 为简单登录流程调用 diagram-generator（浪费回合）

## 与落地版同步

模式 C 新增功能若改变主路径，必须同步更新 §3 图；模式 B 评审时检查动线与功能清单是否一致。
