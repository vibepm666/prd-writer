# 启动前：先读已有材料

用户 **@ 文件**、粘贴文档，或工作区 **`docs/`** 下已有相关稿时，**必须先读再问诊**，不得以空上下文从零重写。

## 1. 扫描并读取

按用户指向或文件名匹配（存在则读，无则跳过）：

| 类型 | 常见路径 |
| --- | --- |
| 已有 PRD / 概念稿 | `docs/*PRD*.md`、`*概念版*.md` |
| 需求与规划 | `docs/requirements.md`、`docs/roadmap.md` |
| 调研与画像 | `docs/*market*`、`*research*`、`*persona*`、`*用户*` |
| 用户 @ 的路径 | 用户消息中明确给出的任意 `.md` / `.txt` |

## 2. 摘录真源

将已有结论整理为 **「已有输入摘要」**（写入概念版顶部，或口头复述），至少覆盖：

- 产品定位与目标用户
- 产品形态与阶段策略
- MVP / V1.0 边界与核心功能方向
- 已明确的「不做什么」
- 仍待确认的问题

## 3. 三视角诊断（仅补缺口）

已写清的角度 **不再重复追问**。每次只问 1–2 个问题，直到用户 / 商业 / 开发三视角均有基本答案。问题库见 `diagnosis-guide.md`。

## 4. 模式判断

| 输入情况 | 模式 |
| --- | --- |
| 完整 PRD，用户要评审或改进 | **模式 B** |
| 有部分稿，或只加/只改某模块 | **模式 C** |
| 仅有调研 / 想法，无 PRD | **模式 A**（诊断从摘要起步） |

## 5. 与第零步的衔接

满足以下任一条件时，**豁免第零步开场问句**，直接进入对应模式：

- 用户已 @ / 提供完整 PRD（→ 模式 B）
- 用户已指明「只加 / 只改某模块」且提供了文档（→ 模式 C）
- 已执行本文件且信息足够判断模式与方向（→ 对应模式）

否则执行 `SKILL.md` 第零步开场问句。
