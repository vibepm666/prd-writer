## 概念版片段（`docs/2026-06-25-专注计时-概念版.md`）

```markdown
# FocusFlow 专注计时（暂定）

## 一句话定位
> 这是一个给需要深度工作的知识工作者用的网页 Web App，帮他们减少分心、记录专注时长。
> 与手机自带计时器相比，核心差异是任务绑定、干扰屏蔽提示与每日统计。

## 产品形态
- **当前选型**：网页 Web App
- **选择理由**：开发快、桌面端为主、无需应用商店审核
- **阶段策略**：验证后再考虑浏览器扩展

## 目标用户
- 独立开发者、设计师等需要长时间专注的人
- 痛点：计时与任务脱节、容易被消息打断、看不到专注趋势

## 核心功能方向
- 开始/结束专注会话并绑定任务名
- 今日专注统计与简单历史
- 专注中全屏提示与浏览器通知（可选）

## 不做什么
- 不做团队协作、不做 Pomodoro 社交排行
```

## 落地版片段（§5 单功能示例）

### 5.1 开始专注会话

**功能描述**：用户输入任务名后开始计时；计时中页面显示剩余/已用时长，可暂停或结束。

**触发条件**：首页点击「开始专注」且任务名非空（或用户确认使用默认任务名）。

**交互细节**：

| 场景 | 处理 |
| --- | --- |
| 操作反馈 | 点击开始后按钮 loading 300ms 内进入计时页 |
| 危险操作确认 | 结束会话时弹窗：「结束本次专注？已专注 25 分钟」 |
| 空状态引导 | 无历史任务时任务输入框 placeholder：「例如：写 PRD」 |
| 失败引导 | 本地存储失败时 toast：「无法保存记录，请检查浏览器权限」 |

**状态清单**：（略，按 prd-template 表填写）

**数据规范**：

| 字段名 | 类型 | 限制 | 必填 | 默认值 |
| --- | --- | --- | --- | --- |
| taskName | string | 1–50 字 | 是 | — |
| startedAt | datetime | ISO8601 | 是 | 当前时间 |
| durationSec | number | ≥0 | 是 | 0 |

**验收标准**：

- [ ] 输入任务名并开始后，计时页显示已用时长且每秒更新
- [ ] 结束会话后，今日统计增加对应分钟数
