# 写后自检清单

正式写入 `docs/` **之前或之后**，逐项勾选。

## 流程与门禁

- [ ] 已执行 `read-first.md`（或有理由跳过：纯口述、无现存稿）
- [ ] 概念版已获用户 **明确确认**（落地版才写 §5 细节）
- [ ] 模式判断正确（A / B / C）
- [ ] 概念版与落地版 **分文件**（或单文件已分第一/第二部分）

## 概念版

- [ ] 路径：`docs/YYYY-MM-DD-<主题>-概念版.md`
- [ ] 含：定位、形态、用户、价值、功能方向、边界、待确认
- [ ] 未展开 §5 级实现细节

## 落地版

- [ ] 路径：`docs/YYYY-MM-DD-<主题>-PRD.md`
- [ ] 文首链回概念版；写明 MVP 范围
- [ ] §3 动线含异常分支；复杂图已按 `diagram-handoff.md` 处理
- [ ] §4.1 含 1 个核心页 ASCII 线框
- [ ] 每个 MVP 🔴 功能有独立 §5.x（🟡⚪ 未越权展开，除非用户要求）
- [ ] `[待补充]` 均标明影响范围，无模糊糊弄
- [ ] §7 已按 `product-type-appendix.md` 补充类型相关项

## 模式 B

- [ ] 已按 `review-checklist.md` 逐项评分
- [ ] 评审报告写入 `docs/YYYY-MM-DD-<主题>-PRD-评审.md`（除非用户不要落盘）
- [ ] 已问用户：直接补全 vs 自行修改

## 模式 C

- [ ] 融合而非覆盖；`【本次更新】` 已标注
- [ ] 动线、功能清单、§5 三者一致
- [ ] 冲突点已请用户裁定

## 通用

- [ ] 终端文案与开发字段说明分开
- [ ] 聊天摘要 + 文件路径已告知用户
